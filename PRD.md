# PRD: 조각투두 — v3

> Claude Code 바이브코딩용 PRD입니다. 구현 대상은 V 0.7이며,
> V 1.0(이어가기)·V 1.3(젤리 보관소·리포트)은 로드맵으로 함께 정의합니다.
> 각 버전은 앞 버전의 가설이 검증된 뒤에 착수합니다.

---

## 0. 서비스 개요

**조각투두**는 할 일을 입력하면 AI가 5분 안에 실행 가능한 미니스텝을 만들어주는, **'일을 미루는 사람을 위한 투두 앱'**이다.

기존 투두앱은 `계획 → 입력 → 시간 설정 → 완료 체크`의 목표지향·완료 중심 구조라서, 결과가 멀고 불확실한 목표를 가진 사용자에게 세 가지 문제를 만든다: ① 할 일이 큰 덩어리로 느껴져 **시작조차 못 하고** ② 완료 유무로만 평가되니 못 하면 **실패로 여겨 자책하고** ③ 당장 얻는 성취가 없어 **지금 할 이유를 못 느낀다.**

조각투두의 해결 전략은 **계획이 아니라 '실행'에 특화된 투두**다. 막막한 할 일의 첫 시작점을 만들어줘(V 0.7), 못 한 일을 실패가 아닌 내일의 더 작은 시작점으로 이어주고(V 1.0), 완료량이 아니라 매일의 시도를 수집형 성취로 돌려준다(V 1.3). 지향점은 완벽한 완수나 생산성 극대화가 아니라, **미루는 사용자가 자책 없이 매일 조금씩 이어가는 습관의 회복**이다.

## 1. 한 줄 정의

**멀고 불확실한 목표 때문에 할 일을 미루는 2030**이 **'시작조차 못 하는 문제'**를 해결하도록 **AI 마이크로 투두 생성(한 줄 입력 → 5분 미니스텝 제안)**을 제공하는 **모바일 우선 웹앱**

## 2. 문제와 타깃

- **문제**: 결과가 멀고 불확실한 목표(취업·이직·성과)를 가진 사람은 할 일이 크고 막막하게 느껴져 시작조차 못 하고 미룬다. 미룬 일이 쌓일수록 자책이 커지고, 다음 시도의 문턱은 더 높아진다.

- **타깃 세그먼트**: '의지로만 해야 하는 목표'를 가진 2030
  1. **이직 준비 직장인** — 퇴근 후 남은 에너지로 포트폴리오·자소서를 스스로 관리해야 하는 사람
  2. **취준생** — 마감·감독자 없이 매일의 학습과 지원 활동을 혼자 굴려야 하는 사람
  3. **사이드잡 하는 학생/직장인** — 본업 외 시간에 결과가 불확실한 프로젝트를 이어가야 하는 사람

- **구체적인 한 사람**: 이직 준비 중인 4년차 직장인 J. "포트폴리오 정리하기"를 3주째 투두앱에 적어두기만 했다. 할 일을 볼 때마다 부담스러워 앱을 열지 않게 됐고, "나만 못 하는 것 같다"는 자책 중이다.

## 3. 버전별 핵심 시나리오

### V 0.7 — 첫 시작점 만들기

1. **진입**: 앱 접속 → 주간 내비게이션과 입력창이 있는 홈 (가입·설정 없음)
2. **입력**: "예: 가계부 작성, 운동 가기, 영어 공부 …" placeholder에 할 일 한 줄 입력 → Enter 또는 추가 버튼
3. **AI 처리**: AI가 판단 없이 몸이 바로 움직일 수 있는, 스텝1(진입 동작 1~2분)·스텝2(전략 기반 첫 착수 동작 2~3분)로 역할이 다른 미니스텝 2개 생성 (합계 5분 이내)
4. **결과**: 선택 날짜 리스트 최상단에 할 일 추가 → 펼친 카드에서 첫 미니스텝 체크 → 하단 토스트로 응원 문구
5. **다음 행동**: 남은 미니스텝·원본 할 일 진행, 또는 새 할 일 입력. 필요하면 스텝 편집(원본 할 일 텍스트 포함), 할일 다시 쪼개기, 정렬 변경, 삭제. **생애 첫 할 일 완수 시** 축하 + 별점 피드백 팝업(1회)

**닫히는 루프**: 입력 → 미니스텝 생성 → 첫 스텝 체크 → 응원 피드백까지, 사용자 한 명 안에서 완결.

### V 1.0 — 못 한 일을 내일의 시작점으로

1. **진입**: 하루가 지나 홈 접속 → 못 끝낸 할 일 카드가 남아 있음. 실패 어휘 없이 "다음에 이어서 할까요?" 문구만 노출. 홈 중앙의 **미스테리 젤리('?')**가 진행 상태에 맞는 다음 행동을 제안 (미착수면 "'컴퓨터 앞에 앉기'부터 해보세요!", 미니스텝을 했다면 원본 할 일 착수 제안)
2. **인터랙션**: 못 끝낸 카드를 **오른쪽으로 스와이프** → 'to be continued' 화살표와 함께 내일로 이동
3. **AI 처리 (조각내기 팝업)**: "다음에는 어디부터 할까요?" 팝업 — 미니스텝이 남았으면 남은 스텝 중 선택, 원본 할 일만 남았으면 입력창에 이어갈 지점 입력 → 입력 기준으로 더 작은 할 일과 미니스텝 재생성
4. **결과**: 다음 날 홈에 '이어진 더 작은 첫 스텝' 카드로 노출 → 부담 없이 재착수
5. **다음 행동**: 첫 스텝 체크 → 시도 칭찬 → 3일 내 재착수·완수로 이어지는지 확인 (D3 재착수율)

**닫히는 루프**: 미완수 → 스와이프 이어가기 → 더 작은 첫 스텝 → 다음 날 재착수까지, 자책 없이 완결.

### V 1.3 — 시도를 수집형 성취로

1. **진입**: 미니스텝·할 일을 완수할 때마다 성취가 **젤리 조각**으로 생성됨 (그날의 완수 조합에 따라 콩 → 하트 → 클로바 → 별 → 토끼 왕젤리로 진화, 젤리 중앙에 1~3 카운트)
2. **수집**: 젤리가 내비게이션 바 '기록·리포트 보관소'의 **큰 병** 안에 쌓임
3. **확인**: 보관소 탭 진입 → 병에 모인 젤리 확인, 보기 전환 아이콘으로 **캘린더 뷰**에서 날짜별 성취 확인
4. **결과**: 완벽함이 아니라 꾸준함을 칭찬하는 **동기부여 리포트** 확인 ("이번 주 5일이나 손댔어요" 류의 시도 중심 피드백)
5. **다음 행동**: 리포트를 본 뒤 새 할 일 입력으로 전환 → 동기부여가 실제 행동으로 이어지는지 확인 (리포트 조회 → 새 입력 전환율, D7 재방문)

**닫히는 루프**: 완수 → 젤리 수집 → 리포트 확인 → 새 입력까지, 성취감이 다음 시도로 순환.

## 4. IA (정보 구조 및 아키텍처)

### 화면 지도

MVP(V 0.7)는 **단일 화면(Single Page)** 이다. 별도 라우트 이동 없이, 한 화면 안에서 상태에 따라 블록이 나타나고 사라진다.

```
[홈 화면] (단일 스크롤 화면)
├─ 헤더 (서비스명 + 카피)
├─ 주간 내비게이션 (WeekStrip)      ← 날짜 선택 / 주 이동
├─ 브랜드 마스코트 + 말풍선         ← 캐릭터 머리 위 중앙에 상태 기반 말풍선
├─ 입력 (TodoInput)                 ← 할 일 추가
└─ 할 일 리스트 (TodoList)          ← 선택 날짜의 할 일 카드들 / 드래그 정렬
     └─ 카드 내부: 헤더(할일명+진행 도트+메뉴, 체크 기능 없음) + 체크리스트(미니스텝 2개+원본 할 일,
        각 항목 오른쪽에 체크)가 테두리·구분선 없이 하나의 연속 블록. 미완료 카드는 항상 펼쳐진 상태
        (접기/펼치기 없음). 단, 전체 완료된 할일 카드는 자동 축소되며 탭으로 재펼침
```

- **오버레이 요소**: 응원/마무리 토스트(하단), 생애 첫 할 일 완수 피드백 팝업, 에러 토스트
- **확장 예정 구조**: 내비게이션 바 '홈 / 기록·리포트 보관소' 2탭 — 0.7에서는 홈만 활성, 보관소 탭은 자리만 두고 잠금 (V 1.3에서 오픈). 잠긴 보관소 탭을 탭하면 화면 이동 없이 하단 토스트 노출: "보관소 기능이 열릴 예정이에요. 곧 만나요!"

## 5. 버전별 핵심 기능

### V 0.7 — 시작의 진입장벽 제거 (이번 구현 범위)

**가설**: 할 일을 '판단 없이 5분 안에 몸이 움직일 수 있는 구체적 동작'으로 쪼개 첫 시작점을 제시하면, 착수율이 오른다.

#### 1. 할 일 입력
- 할 일은 한 줄 자유 텍스트로 입력한다 (계획 부담 없는 단일 인풋)
- 최대 50자. 초과 입력은 인풋에서 차단한다
- 공백만 입력한 경우 추가할 수 없다
- Enter 또는 추가 버튼으로 제출한다 (버튼 레이블: '할일 등록')
- 입력창 바로 아래 작고 연한 스타일의 보조 안내 문구를 상시 노출한다: "할일을 쉽게 시작할 수 있도록 작게 조각내어 드릴게요"
- 제출한 할 일은 **현재 선택된 날짜**의 리스트에 추가된다

#### 2. AI 미니 스텝 생성
- 제출 시 AI가 5분 내 실행 가능한 미니 스텝을 **정확히 2개** 생성한다 (개수 고정, 가변 아님)
- **시간 규칙**: 스텝1 1~2분 + 스텝2 2~3분, 합계 5분 이내
- **두 스텝은 역할이 다르다**: 스텝1은 몸을 그 일 앞으로 옮기는 **진입 동작**(단일 동작 또는 이어지는 짧은 동작 2개까지 묶기 허용, 예: "책상에 앉아서 노트북 켜기"). 스텝2는 시작 전략에 맞춰 생성하는 **전략 기반 첫 착수 동작** (전략 상세는 '7-1. 미니스텝 다양화' 참조). 둘 다 생각·계획이 필요한 동작은 금지
- 실패(API 오류·응답 파싱 실패·스텝 조건 위반) 시 토스트 안내 문구와 기본 폴백 스텝 제공
- AI 상세 규칙(입력/출력/프롬프트/폴백)은 '7. AI 동작 정의', '7-1. 미니스텝 다양화' 섹션에 둔다

#### 3. 체크리스트 (미니 스텝 + 원본 할 일)
- 하나의 할 일은 **미니 스텝 2개 + 원본 할 일 1개**로 구성된 체크리스트를 가진다
- 원본 할 일 항목은 **항상 맨 아래**에 위치하며, 다른 스텝과 같은 체크 모양이되 시간 표기가 없다
- 각 항목은 체크/해제할 수 있다
- 미니 스텝 하나라도 체크되면 **하단 토스트로 응원 문구**, 모두 체크되면 **마무리 문구**를 보여준다
- **생애 첫 할 일 완수 시**: 축하 메시지 "첫 할 일을 해냈어요!"와 함께 "할 일을 시작하는 데 도움이 됐나요?"를 묻는 별점(1~5) 피드백 팝업 노출. [알려주기] 버튼으로 제출, X로 응답 없이 닫기 가능. 이 팝업은 첫 할 일 수행 완료 시에만 **한 번** 노출된다 (상세는 '6-3. 마스코트 말풍선 · 생애 첫 완수 별점 피드백 팝업' 참조)
- 메뉴의 **[스텝 편집]**에서 미니 스텝 텍스트와 **원본 할 일 텍스트**를 함께 수정할 수 있다 (스텝 수동 추가 기능은 제공하지 않음 — 항상 2개 고정이므로). 원본 할 일 텍스트는 카드 헤더(할일명)와 체크리스트 맨 아래 항목이 같은 데이터를 참조하므로 하나를 고치면 둘 다 갱신된다. 입력 규칙은 최초 입력과 동일(최대 50자, 공백만 입력 불가 — 공백만 남기면 저장이 막히고 "할 일 내용을 적어주세요" 안내와 함께 [편집 끝내기]가 비활성화된다)
- **[편집 끝내기]** 시 원본 할 일 텍스트가 실제로 바뀐 채로 저장되면, 미니 스텝 처리 방식을 묻는 얼럿이 뜬다: "할 일이 바뀌었어요. 미니스텝은 어떻게 할까요?" + **[텍스트만 수정]**(주요 버튼, 기존 미니 스텝 그대로 유지) / **[다시 쪼개기]**(보조 버튼, 아래 [할일 다시 쪼개기]와 동일한 로직으로 미니 스텝만 새로 생성 — 미니 스텝 체크 상태는 초기화되고 원본 할 일 체크 상태는 유지). 이 얼럿은 X·백드롭 닫기가 없고 두 버튼 중 하나를 반드시 선택해야 하며, 어느 쪽을 선택해도 텍스트 수정 자체는 이미 확정된 상태라 취소되지 않는다
- **[할일 다시 쪼개기]**(카드 메뉴)로 미니 스텝을 새로 생성할 수 있다 (원본 할 일은 유지된 채 미니 스텝만 교체, 직전 사용 전략은 재샘플링 후보에서 제외)

#### 4. 날짜 내비게이션
- 주간 스트립에서 날짜를 선택해 그 날짜의 할 일을 조회/추가한다
- 이전/다음 주로 주 범위를 이동할 수 있다. **주를 이동해도 선택 날짜는 그대로 유지된다** (주 스트립만 앞뒤로 넘어감)
- 할 일은 날짜별로 분리 저장된다
- **과거·미래를 포함해 어느 날짜에서든 할 일을 추가할 수 있다** (날짜 제한 없음)

#### 5. 순서 변경 (드래그 정렬)
- 리스트에서 항목을 드래그해 순서를 바꾼다
- **새로 추가한 할 일은 항상 최상단**에 들어간다. 그 외 순서는 사용자의 드래그 결과를 보존한다
- **완료된 항목도 위치를 바꾸지 않는다** (자동 정렬/이동 없음)

#### 6. 삭제
- 카드 메뉴에서 할 일 단위로 삭제한다

#### 7. 퍼널 이벤트 로깅

- 보조 이벤트: 할일 다시 쪼개기, 스텝 편집, 삭제, `first_feedback`(생애 첫 완수 별점 피드백, `rating` 1~5 또는 'dismissed')

**0.7에서 안 하는 것 (Won't)**: 이어가기·내일로 넘기기(1.0) / 미스테리 젤리(1.0) / 젤리 보관소·리포트·캘린더 보기(1.3) / 회원가입·로그인 / 푸시 알림·리마인더 / 마감일·시간 설정 / 반복 할 일·카테고리·태그 / 다크모드·온보딩 튜토리얼 / 스텝 수동 추가(편집만 지원)

### V 1.0 — 미완수를 '내일의 시작점'으로 연결

**가설**: 못 한 일을 '실패'가 아니라 '내일의 더 작은 시작점'으로 이어주면, 3일 내 재착수·완수율이 오른다.

1. **부담 없는 미완수 문구** — 당일 할 일을 못 끝냈을 때 "다음에 이어서 할까요?" 같은 전방향 언어로만 상태 표현 (실패 어휘 없음)
2. **미스테리 젤리** — 할 일을 생성하면 '?' 모양 미스테리 젤리가 홈 화면 중앙에 생성. 진행 상태에 따라 다음 행동을 제안:
   - 미착수 상태 → "'컴퓨터 앞에 앉기'부터 해보세요!" 식으로 첫 미니스텝 제안
   - 미니스텝 완수 상태 → 원본 할 일 착수 제안
3. **이어하기 (조각내기 팝업 인터랙션)** —
   - 못 끝낸 카드를 **오른쪽으로 스와이프**하면 'to be continued' 화살표와 함께 내일로 이동
   - "다음에는 어디부터 할까요?" 팝업에서 이어갈 지점 선택:
     - 미니스텝이 남았으면 → 남은 미니스텝 중 선택
     - 원본 할 일만 남았으면 → 입력창 제공 → 입력 내용에 맞는 할 일·미니스텝 재생성
   - 다음 날 홈에 이어진 할 일이 '내일의 더 작은 첫 스텝'으로 노출

### V 1.3 — 시도의 성취를 수집으로 시각화

**가설**: 완료량이 아니라 매일의 시도를 즉각적인 작은 성취 피드백(수집)으로 주면, 7일 내 재방문·재시도가 유지된다.

1. **젤리 조각 수집 (보관소)** — 해낸 일이 젤리 조각으로 만들어져 화면 속 큰 병에 쌓임. 내비게이션 바 '기록·리포트 보관소' 탭 오픈
2. **성취량에 따른 젤리 진화** — 해낸 양의 조합에 따라 젤리 모양이 달라짐:
   - **1단계 · 콩 젤리**: 미니스텝 1개 이상 완수
   - **2단계 · 하트 젤리**: 미니스텝 2개 완수 + 할 일 1개 조각내기
   - **3단계 · 클로바 젤리**: 미니스텝 2개 완수 + 할 일 1개 완수
   - **4단계 · 별 젤리**: 미니스텝 2개 완수 + 할 일 1개 완수 + 다른 할 일의 미니스텝 1개 이상 완수
   - **5단계 · 토끼 왕젤리**: 미니스텝 2개 완수 + 할 일 1개 완수 + 다른 할 일의 미니스텝 2개 완수 + 할 일 1개 완수
3. **젤리 카운트** — 젤리 중앙에 한 일 카운트 표시. 숫자는 미니스텝 2개와 원본 할 일 완수만 포함해 1~3
4. **캘린더 보기** — 보관소에서 보기 전환 아이콘으로 캘린더 뷰 제공, 날짜별 젤리(성취) 확인
5. **동기부여 리포트** — 완벽함이 아니라 꾸준함을 칭찬하는 리포트 (참고 레퍼런스: 오운완, 다이어트 앱)

> **열린 결정 (1.3 착수 전 확정)**: 젤리 생성 단위가 '하루 1개(그날 성취 조합으로 모양 결정)'인지 '할 일 1개당 1개'인지. 중앙 카운트 1~3은 할 일 단위 기준이므로, 4~5단계처럼 여러 할 일에 걸친 조건과 카운트 표기 방식의 정합을 맞춰야 함.
>
> **열린 결정 — 하루 단위 난이도/소감 기록**: 하루 단위로 그날 할 일이 어땠는지 기록하는 기능이 별도로 필요한지, 필요하다면 수집 방식이 무엇인지 미정 (별점 방식은 아님 — 그건 생애 첫 완수 피드백 팝업에서 이미 사용). 리포트 설계와 함께 확정하고, 날짜별 회고 데이터를 캘린더 뷰·동기부여 리포트에서 어떻게 활용할지도 같이 정할 것.

## 6. 화면 구성 (V 0.7)

IA의 단일 화면 구조를 따른다. 블록별 구성 요소:

- **헤더**: 서비스명 + 한 줄 카피
- **주간 내비게이션 (WeekStrip)**: 이번 주 7일 표시, 날짜 탭 선택, 좌우로 주 이동 (선택 날짜 유지). 상단 표기는 "n년 n월 n일"로 **선택된 날짜** 기준(주 이동 시에도 선택 날짜는 유지되므로 표기도 그대로 유지, 주만 앞뒤로 넘어감). 날짜 사이 간격은 `spacing/8` 명시 토큰(기존엔 `justify-between` 자동 분배라 토큰이 없었음, `grid grid-cols-7`로 전환하며 명시적 gap 부여). 상단 이전/다음 주 화살표와 날짜 표기 사이 간격도 같은 이유로 `spacing/24` 명시 토큰 사용(기존엔 `justify-between`이라 토큰이 없었음, `justify-center` + 명시적 gap으로 전환)
- **브랜드 마스코트 + 말풍선**: 주간 내비게이션 바로 아래, 화면 중앙 정렬. 캐릭터 이미지(120×120) 머리 위 중앙에 말풍선이 항상 떠 있다. 상세는 "6-3. 마스코트 말풍선" 참조
  - **신규 등록 로딩 연출(큰 피드백)**: 신규 할일 등록 → 미니스텝 생성 중 캐릭터가 리듬감 있게 점프(loop)하고 말풍선은 "할일을 쪼개고 있어요"를 노출한다. 생성이 완료되면 점프가 멈추고 말풍선은 순간 반응 ①("좋아요, 이만큼 작아졌어요" 등)로 전환된다. 캐릭터 점프는 **신규 등록에만** 쓰고 재생성에는 쓰지 않는다. (모션 스펙은 design-system md 7-1 참조)
- **입력 (TodoInput)**: 입력창(placeholder: "예: 가계부 작성, 운동 가기, 영어 공부 …", 최대 50자) + '할일 등록' 버튼. 입력창 아래 작고 연한 보조 안내 문구("할일을 쉽게 시작할 수 있도록 작게 조각내어 드릴게요") 상시 노출 + 인포 아이콘(탭 시 툴팁으로 "할 일을 적을 때, '네일'보다 '네일 받기'처럼 적으면 더 정확하게 할 일을 쪼갤 수 있어요." 노출)
- **할 일 리스트 제목**: 입력 영역과 리스트 영역을 구분하는 왼쪽 정렬 소제목. 선택 날짜가 오늘이면 "오늘 할 일", 아니면 "n월 n일 할 일" + 뒤에 해당 날짜 할 일 개수 "N개" 표기 (예: "오늘 할 일 3개")
- **할 일 리스트 (TodoList)**: 선택 날짜의 할 일 카드들. 카드 하나당 외곽 테두리 1개, 내부 구분선 없음. 드래그 정렬, 카드 메뉴(스텝 편집 · 할일 다시 쪼개기 · 삭제). 완료 항목은 위치 유지
  - **카드 구성**: 헤더(할일명+진행 도트+메뉴 — 체크 기능 없음)와 체크리스트(미니 스텝 2개[시간 표기] + 원본 할 일[맨 아래, 시간 표기 없음], 각 항목 체크는 오른쪽 배치)가 테두리·구분선 없는 하나의 연속 블록. 항목 간 구분은 여백(padding/margin)만 사용
  - **진행 도트**: 헤더의 할일명 옆에 작은 원 3개(미니스텝 2개+원본 할 일 1개 = 3칸). 체크된 항목 수만큼 채워진 원으로 표시
  - **접기/펼치기**: 미완료 카드는 항상 펼쳐진 상태(접기 없음). **전체 완료(3/3)된 카드는 자동으로 축소**되어 할일명+진행 도트만 남고 체크리스트는 숨김. 축소된 카드를 탭하면 다시 펼쳐지고, 펼친 상태에서 체크 해제 가능(해제 시 다시 미완료 규칙에 따라 항상 펼침으로 전환)
  - **편집 모드**: 메뉴의 [스텝 편집] 진입 시 미니 스텝 2개와 체크리스트 맨 아래 원본 할 일 항목이 모두 인풋으로 전환(헤더는 읽기 전용 유지, 원본 할 일과 같은 데이터라 실시간 동기화됨). 원본 할 일 인풋이 공백만 남으면 인풋 아래 경고 문구("할 일 내용을 적어주세요")를 노출하고 [편집 끝내기] 버튼을 비활성화. [편집 끝내기] 시 원본 할 일 텍스트가 실제로 바뀌었으면 재생성 얼럿(바텀시트, X·백드롭 닫기 없음)이 뜬다 — 주요 버튼 [텍스트만 수정](미니 스텝 유지) / 보조 버튼 [다시 쪼개기](미니 스텝만 재생성, 체크 초기화·원본 체크 유지)
  - **재생성 로딩 연출(작은 피드백)**: 카드 메뉴 [할일 다시 쪼개기] 또는 수정 후 [다시 쪼개기]로 미니스텝을 재생성하는 동안, 캐릭터 점프 대신 **해당 카드의 체크리스트 자리에 인라인 로더**(진행 도트와 같은 6px brand 원 3개 순차 펄스 + "쪼개는 중이에요")를 표시한다. 완료되면 로더가 사라지고 새 미니스텝을 표시한다. (모션 스펙은 design-system md 7-1 참조)
- **토스트**: 첫 체크 응원 문구 / 전체 체크 마무리 문구 / 에러 안내 / 보관소 탭(잠금) 탭 시 "보관소 기능이 열릴 예정이에요. 곧 만나요!" (모두 하단). 콘텐츠 폭 전체를 채우는 넓은 가로폭 + 다크 서피스(`background/inverse`)로 눈에 띄게 표시
- **팝업 · 생애 첫 할 일 완수**: 축하 메시지 "첫 할 일을 해냈어요!" + "할 일을 시작하는 데 도움이 됐나요?" + 별점(1~5) + [알려주기] / X 닫기 (1회만, 상세는 6-3 참조)
- **상태 · 빈 리스트**: "해야 할 일을 하나만 적어봐요."

### 6-2. 라이팅 톤: "텐션 중간 친구"

다정하지만 부담스러운 응원단은 아닌 친구. 해요체. 사용자는 이미 자책 중이므로, 하이텐션 응원("오늘도 파이팅! 💪")도, 중립적 시스템 언어("3일째 미완료입니다")도 모두 독이다.

**원칙 1 — 실패 어휘 금지**: "미완료", "실패", "달성률", "밀린", "지연" 등 부정·평가 어휘를 UI 어디에도 쓰지 않는다. 못 한 상태는 "다음에 이어서 할까요?", "내일 더 작게, 다시 이어서 해봐요" 같은 전방향 언어로만 표현한다.

**원칙 2 — 성취의 재정의를 카피가 직접 수행**: 체크 후 문구는 "완료!"가 아니라 "시작이 반이에요. 이대로 잘하고 있어요". '시도 = 성공' 프레임을 모든 접점에서 반복한다.

**원칙 3 — 명령형 대신 청유형 + 낮은 허들 동사**: "완료" 대신 '했어요', "시작하세요" 대신 "5분만 해볼까요?"

> 제출 버튼 레이블은 '할일 등록'으로 변경됨(기능 명확성 우선 — 원칙 3의 청유형 톤 예외로 명시).

**금지어 → 대체어**

| 금지 | 대체 |
| --- | --- |
| 미완료, 실패 | 다음에 이어서 할까요? / 내일 더 작게 이어가요 |
| 달성률 00% | (표기 자체를 하지 않음) |
| 완료! | 했어요 / 시작이 반이에요 |
| 할 일을 입력하세요 | 예: 가계부 작성, 운동 가기, 영어 공부 … |
| 다시 시도하세요 (에러) | 잠깐 삐끗했어요. 한 번만 다시 눌러줄래요? |

**접점별 카피 예시**: 입력 placeholder "예: 가계부 작성, 운동 가기, 영어 공부 …" / 입력 인포 툴팁 "할 일을 적을 때, '네일'보다 '네일 받기'처럼 적으면 더 정확하게 할 일을 쪼갤 수 있어요." / 신규 생성 로딩 — 말풍선 "할일을 쪼개고 있어요", 버튼 "쪼개는 중이에요" / 재생성 인라인 로딩 "쪼개는 중이에요" / 생성 직후 "5분만 해볼까요?" / 첫 체크 토스트 "시작이 반이에요. 이대로 잘하고 있어요" / 전체 체크 토스트 "오늘 몫은 충분해요" / 첫 할 일 완수 팝업 "첫 할 일을 해냈어요! 이 기능이 일을 시작하는 데 도움이 됐나요?" / 빈 리스트 "해야 할 일을 하나만 적어봐요." / 에러 토스트 "잠깐 삐끗했어요. 한 번만 다시 눌러줄래요?"

### 6-3. 마스코트 말풍선

캐릭터 머리 위 중앙에 항상 떠 있는 말풍선. 좌우 `spacing/12`, 상하 `spacing/8` 패딩, `radius/12`, 배경 `background/surface`. 꼬리는 말풍선 아래쪽 중앙에서 마스코트 머리를 향한다.

**상태 판정** (선택 날짜의 리스트 전체 기준):
- 빈 리스트: 할 일 0개
- 미착수: 할 일은 있으나 체크된 항목 0개
- 진행 중: 체크된 항목 1개 이상, 전체 완수는 아님
- 전체 완수: 모든 할 일의 모든 항목(미니스텝+원본) 체크됨

**상시 상태 문구** — 6-2 접점별 카피에서 이미 승인된 문구를 재사용:

| 상태 | 문구 | 출처 |
|---|---|---|
| 빈 리스트 | "해야 할 일을 하나만 적어봐요." | 6-2 빈 리스트 카피 그대로. 고정 문구(로테이션 없음) |
| 미착수 | "5분만 해볼까요?" | 6-2 생성 직후 카피 재사용 |
| 진행 중 | "시작이 반이에요. 이대로 잘하고 있어요" / "그 한 걸음이 제일 무거운 거였어요" / "좋아요, 흐름 탔어요" / "오늘도 한 걸음씩 나아가고 있어요" (4종 풀에서 로테이션) | ② 미니스텝 체크 시 순간 반응 카피 풀 재사용 + 신규 1종. 미니스텝 체크 시점마다 함께 로테이션됨(고정 문구 아님) |
| 전체 완수 | "오늘 몫은 충분해요" | 6-2 전체 체크 토스트 카피 그대로 |

**순간 반응 (행동 이벤트별)** — 행동 직후 3초간 노출 후 상시 상태 문구로 복귀. 이벤트별 카피 풀에서 매번 무작위 선택하되, 직전과 같은 문구는 연속으로 노출하지 않는다.

| 트리거 | 시점 | 카피 풀 |
|---|---|---|
| ① 미니스텝 생성 직후 | 할일 등록 또는 할일 다시 쪼개기로 미니스텝 생성 완료 시 | "좋아요, 이만큼 작아졌어요" / "이 정도면 해볼 만하죠?" / "첫 번째 거, 5분이면 돼요" |
| ② 미니스텝 체크 시 | 체크리스트 항목 체크(할일 통째 완수가 되는 체크는 제외 → ③으로) | "시작이 반이에요. 이대로 잘하고 있어요" / "그 한 걸음이 제일 무거운 거였어요" / "좋아요, 흐름 탔어요" |
| ③ 할일 통째 완수 시 | 원본 할 일까지 포함해 한 할일의 모든 항목 체크 (단, 생애 첫 완수는 제외 → 별점 팝업으로 대체, 오늘 할 일 전체 완수가 되는 경우는 제외 → ⑤로) | "하나를 통째로 해냈어요" / "미루던 게 방금 끝났어요" |
| ⑤ 오늘 할 일 전체 완수 시 | 오늘 리스트의 모든 할 일 · 모든 항목(미니스텝+원본)이 체크됨 (③보다 우선) | "오늘 몫, 전부 해냈어요" / "마지막 조각까지 끝났어요. 이제 진짜 쉬어요" / "오늘은 미룬 게 하나도 없네요" |

**생애 첫 완수 별점 피드백 팝업** — 기존 "도움됐어요/도움이 안 됐어요" 2버튼 방식을 대체.
- 노출 조건: 생애 첫 할 일 통째 완수(원본까지 전부 체크) 시 1회만. localStorage 플래그로 재노출 방지
- 이 경우 ③ 순간 반응은 생략하고 팝업을 대신 노출, 팝업이 닫히면 상시 상태 문구로 전환
- 구성: 캐릭터(마스코트 화면 상단에 항상 있음) + 축하 메시지 "첫 할 일을 해냈어요!" + 질문 "할 일을 시작하는 데 도움이 됐나요?" + 별 5개(탭으로 1~5 선택, 시각 40px·탭 타겟 48px) + 선택 라벨 + [알려주기] 버튼, X 아이콘으로 응답 없이 닫기 가능
- 별 선택 시 별 아래에 점수별 라벨 노출(선택 전엔 빈 자리 높이 확보): ★1 "잘 모르겠어요" / ★2 "조금 아쉬워요" / ★3 "나쁘지 않아요" / ★4 "약간 도움됐어요" / ★5 "많이 도움됐어요!" (스타일·여백 상세는 design-system md "생애 첫 완수 별점 팝업" 참조)
- 데이터: localStorage에 `{ rating: 1~5 또는 'dismissed' }` 저장
- Mixpanel: `first_feedback` 이벤트, `rating` 프로퍼티(1~5 또는 'dismissed')

## 7. AI 동작 정의 (V 0.7)

- **입력**: 자유 텍스트 한 줄, 최대 50자 (초과는 입력 단계에서 차단). 신규 입력과 '할일 다시 쪼개기' 모두 동일한 규칙 적용
- **출력**: 정해진 구조의 JSON 형식으로 고정. 다른 설명 문장이나 마크다운 없이 아래 **3개 필드를 이 순서로** 담아 반환한다.
  1. **analysis** — 이 할 일을 실제로 하는 데 필요한 장소·물건(도구)·첫 물리적 행동을 각 한 줄로 서술. **UI에는 표시하지 않는 내부 필드**이며, 클라이언트는 파싱 후 버린다. 스텝 생성 품질을 위한 사고 과정 강제용
  2. **strategy** — 선택한 시작 전략 1개
  3. **ministeps** — 미니스텝 목록. 각 미니스텝은 두 가지 값을 가진다 — 스텝 문구(40자 이내의 물리적 동작 서술), 예상 소요 시간(분 단위). 미니스텝 개수는 **정확히 2개** (1개나 3개 이상은 조건 위반)
- **시간 규칙**: 스텝1 1~2분, 스텝2 2~3분, 전체 합계 5분 이내
- **스텝 작성 규칙**: 두 스텝은 역할이 다르다
  - **스텝1 · 진입 동작 (1~2분)**: 몸을 그 일 앞으로 옮기는 물리적 동작. 단일 동작 또는 이어지는 짧은 동작 2개까지 묶기 허용(예: "책상에 앉아서 노트북 켜기"). 묶어도 시간 안에 끝나야 하고, 생각·계획이 필요한 동작은 묶기 금지
  - **스텝2 · 전략 기반 첫 착수 동작 (2~3분)**: '7-1. 미니스텝 다양화'에서 전달받은 시작 전략 후보 중 할 일에 가장 적합한 1개를 선택해, 그 전략에 맞는 구체적 동작으로 생성
- **프롬프트 방향**:
  - 역할: 행동경제학 기반 실행 코치. '판단이나 계획 없이 몸이 바로 움직일 수 있는 물리적 첫 동작'만 제안. 스텝2는 전달된 전략 후보 중 1개를 선택하는 방식으로 동작
  - **절차형 자가분석 강제**: 스텝을 만들기 전에 반드시 순서대로 답하게 한다 — **이 일은 어디서 하는가 → 무엇이 필요한가 → 몸이 하는 첫 동작은 무엇인가** (= analysis 필드). 스텝은 이 분석에 나온 장소·물건·행동만 사용해 작성한다. **분석에 등장하지 않은 물건이나 장소를 스텝에 쓰지 말 것.** (기존에 시도했던 "도메인 4종 분류 + 도메인별 few-shot 예시 확장" 방식은 예시가 늘어날수록 다루지 못하는 할 일이 계속 남는 구조라 폐기하고, 이 자가분석 방식으로 대체함)
  - 톤: 해요체, 담백하고 다정. 6-2 라이팅 원칙 준수
  - 금지: 조언·설교·동기부여 문구 생성 금지, 실패 어휘 금지, 시간 규칙 초과 스텝 금지, "계획 세우기"류의 추상 스텝 금지, **analysis에 없는 물건·장소를 스텝에 사용하는 것 금지**
- **할일 다시 쪼개기**: 원본 할 일 텍스트를 기준으로 미니스텝만 새로 생성해 교체한다. 원본 할 일 항목과 그 체크 상태는 유지. **직전에 사용한 전략은 재샘플링 후보에서 제외**한다 (7-1 참조)
- **실패 처리**:
  - API 오류·타임아웃(8초): 에러 토스트 "잠깐 삐끗했어요. 한 번만 다시 눌러줄래요?" + 재시도
  - 응답 형식 파싱 실패: 1회 자동 재요청 → 재실패 시 위 에러 토스트
  - **스텝/필드 조건 위반**: 아래 중 하나라도 해당하면 1회 자동 재요청 → 재실패 시 범용 폴백 스텝 제공 — "일단 그 일과 관련된 창 하나만 열어보기" (2분, 전략 없이 스텝1 성격의 폴백 1개만 제공)
    - `ministeps` 개수가 2개가 아님
    - 시간 규칙 위반 (스텝1 1~2분·스텝2 2~3분·합계 5분 범위 밖)
    - 최상위 필드(`analysis`/`strategy`/`ministeps`) 중 하나라도 누락
    - `analysis`의 하위 필드(`location`/`materials`/`firstAction`) 중 하나라도 누락 또는 빈 문자열
  - **analysis-스텝 정합성(analysis에 없는 물건·장소를 스텝이 쓰는 것)은 프롬프트 지시로만 통제하며, 클라이언트 검증 대상이 아니다.** 자동 재요청은 위의 결정론적 규칙(미니스텝 개수 2개, 시간 규칙, JSON 파싱·필드 누락)에만 적용한다

## 7-1. 미니스텝 다양화 — 전략 기반 2단계 생성

같은 할 일을 입력해도 "앉기·열기·파일 찾기"류로 수렴하지 않도록, 스텝을 바로 생성하지 않고 시작 전략을 먼저 정한 뒤 그 전략에 맞는 스텝을 생성하는 2단계 구조를 쓴다.

**동작 흐름**:
1. 클라이언트가 시작 전략 풀(7종) 중 **무작위 3개를 샘플링**해 AI에 후보로 전달
2. AI가 먼저 장소·물건·첫 동작을 스스로 분석(analysis 필드, 7번 참조)한 뒤, 할 일에 가장 적합한 **1개 전략을 선택**하고, 그 전략과 분석 결과에 맞는 스텝2(전략 기반 첫 착수 동작)를 생성
3. 응답에 **선택한 전략**을 포함해 반환 (데이터 저장·Mixpanel 로깅에 사용). analysis 필드는 로깅하지 않고 파싱 후 버림
4. **할일 다시 쪼개기** 시에는 직전에 사용한 전략을 후보에서 제외하고, 나머지 6종 중 3개를 다시 무작위 샘플링

**시작 전략 풀 (7종)** — 스텝2(첫 착수 동작)에 적용:

| 전략 | 예시 (입력: "포트폴리오 만들기") |
|---|---|
| 자료 모으기 | 프로젝트 이미지 3장 찾기 |
| 아주 작은 실행 | 소개 문장 한 줄만 적기 |
| 결과물 일부 만들기 | 대표 화면 1개 캡처하기 |
| 이전 작업 이어보기 | 마지막으로 수정했던 페이지 열기 |
| 방해 요소 제거 | 바탕화면에서 불필요한 창 닫기 |
| 예시 하나 찾기 | 참고 포트폴리오 하나만 열어보기 |
| 검토부터 시작 | 기존 포트폴리오 2분만 훑어보기 |

### Few-shot 예시 — 형식 앵커용 2개만 유지

> **폐기 이력**: 이전엔 "도메인 4종 분류 + 도메인별 few-shot 예시 확장"으로 컴퓨터 편향 문제에 대응했으나, 예시를 늘려도 다루지 못하는 할 일이 항상 남는 구조라 폐기함. 대신 AI가 매 요청마다 스스로 장소·물건·행동을 분석하는 **자가분석 구조**(analysis 필드, 7번 참조)로 전환. few-shot은 이제 도메인 커버리지용이 아니라 **출력 형식을 보여주는 앵커 역할만** 한다.

**예시는 형식 참고용이며, 여기 나온 동작을 그대로 재사용하지 않는다.** 서로 최대한 다른 도메인 2개로 구성:

입력: "자소서 쓰기"
```json
{
  "analysis": {
    "location": "책상 앞, 노트북 화면",
    "materials": "노트북, 자소서 문서 파일",
    "firstAction": "책상에 앉아서 노트북 켜기"
  },
  "strategy": "이전 작업 이어보기",
  "ministeps": [
    { "text": "책상에 앉아서 노트북 켜기", "minutes": 2 },
    { "text": "마지막으로 썼던 문단 다시 읽어보기", "minutes": 2 }
  ]
}
```

입력: "운동 가기"
```json
{
  "analysis": {
    "location": "옷장 앞, 운동하는 장소",
    "materials": "운동복, 운동화",
    "firstAction": "운동복으로 갈아입기"
  },
  "strategy": "아주 작은 실행",
  "ministeps": [
    { "text": "운동복으로 갈아입기", "minutes": 2 },
    { "text": "스트레칭 한 동작만 하기", "minutes": 3 }
  ]
}
```

## 8. 기술 스택

- **프레임워크**: Next.js (App Router), 모바일 우선 반응형, 단일 페이지
- **DB**: **Supabase** — 서버 데이터 연결이 필요한 시점부터 도입. 0.7은 기기 내 저장(localStorage)으로 충분하며, 1.0의 이어가기(날짜 간 데이터 이동)와 1.3의 보관소(장기 누적 기록)부터 Supabase 익명 세션 기반으로 전환. 단, 0.7 설계 시 데이터 구조를 Supabase 테이블로 옮기기 쉽게 잡아둘 것
- **AI API**: Anthropic Claude API (짧은 구조화 출력이므로 저비용 모델로 충분). API 키는 서버 라우트에서만 사용, 클라이언트 노출 금지
- **분석**: Mixpanel
- **배포**: Vercel

**Mixpanel 이벤트 (0.7)**: 할일 입력 / 미니스텝 생성(`strategy` 프로퍼티 포함 — 전략별 착수율 비교용) / 첫 스텝 체크 / 전체 완수 — 4단계 퍼널. 보조: 할일 다시 쪼개기, 스텝 편집, 삭제, `first_feedback`(`rating` 프로퍼티 1~5 또는 'dismissed')

**익명 사용자 식별**: 로그인이 없으므로 클라이언트에서 익명 ID를 생성해 localStorage에 저장하고(앱 최초 진입 시 1회 생성, 이후 재사용), Mixpanel `identify()` 호출에 사용한다. 이 ID는 V1.0 Supabase 전환 시의 익명 세션 ID와 동일하게 재사용 가능하도록 설계한다
